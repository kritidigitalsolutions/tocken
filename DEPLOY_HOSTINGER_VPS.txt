Hostinger VPS Deployment Guide for tocken_app_Admin_panel
=========================================================

This guide deploys:
1) Backend API (Node/Express) on PM2
2) Admin Frontend (frontend - CRA build)
3) Public Website (website - Vite build)
4) Nginx reverse proxy + SSL

Assumptions
-----------
- VPS OS: Ubuntu 22.04/24.04
- Domains:
  - yourdomain.com (public website)
  - admin.yourdomain.com (admin panel)
  - api.yourdomain.com (backend API)
- Project path on VPS: /var/www/tocken_app_Admin_panel


STEP 1) VPS Basic Setup
-----------------------
SSH login:
ssh root@YOUR_VPS_IP

Update system:
apt update
apt upgrade -y

Install base tools:
apt install -y curl git nginx ufw build-essential

Install Node.js with NVM (Node 22):
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
source ~/.nvm/nvm.sh
nvm install 22
nvm use 22
node -v
npm -v

Install PM2:
npm install -g pm2


STEP 2) Clone Project
---------------------
mkdir -p /var/www
cd /var/www
git clone https://github.com/kritidigitalsolutions/tocken_app_Admin_panel.git
cd /var/www/tocken_app_Admin_panel


STEP 3) Backend Setup (Node/Express)
------------------------------------
cd /var/www/tocken_app_Admin_panel/backend
npm install

Create backend env file:
nano .env

Paste (example):
NODE_ENV=production
PORT=5000
MONGO_URI=mongodb+srv://USERNAME:PASSWORD@cluster.mongodb.net/DB_NAME
JWT_SECRET=your_super_secure_secret

FIREBASE_PROJECT_ID=...
FIREBASE_PRIVATE_KEY_ID=...
FIREBASE_PRIVATE_KEY="-----BEGIN PRIVATE KEY-----\n...\n-----END PRIVATE KEY-----\n"
FIREBASE_CLIENT_EMAIL=...
FIREBASE_CLIENT_ID=...
FIREBASE_CERT_URL=...
FIREBASE_STORAGE_BUCKET=...

EMAIL_USER=...
EMAIL_PASS=...
RAPIDSMS_API_KEY=...
RAPIDSMS_SENDER_ID=...
GOOGLE_MAPS_API_KEY=...

PHONEPE_ENV=SANDBOX
PHONEPE_CLIENT_ID=...
PHONEPE_CLIENT_SECRET=...
PHONEPE_CLIENT_VERSION=1
PHONEPE_REDIRECT_URL=https://yourdomain.com/payment-status

Start backend with PM2:
pm2 start server.js --name tocken-backend
pm2 logs tocken-backend --lines 100

Enable PM2 on restart:
pm2 startup
pm2 save


STEP 4) Build Admin Frontend (frontend)
---------------------------------------
cd /var/www/tocken_app_Admin_panel/frontend
npm install

Create env file:
nano .env

Add:
REACT_APP_API_URL=https://api.yourdomain.com/api

Build:
npm run build

Output folder:
/var/www/tocken_app_Admin_panel/frontend/build


STEP 5) Build Public Website (website)
--------------------------------------
cd /var/www/tocken_app_Admin_panel/website
npm install

Create env file:
nano .env

Add:
VITE_API_BASE_URL=https://api.yourdomain.com
VITE_GOOGLE_MAPS_API_KEY=your_google_maps_api_key

Build:
npm run build

Output folder:
/var/www/tocken_app_Admin_panel/website/dist


STEP 6) Nginx Config (3 sites)
------------------------------
Remove default site (optional but recommended):
rm -f /etc/nginx/sites-enabled/default

A) Public website config:
nano /etc/nginx/sites-available/website.conf

Paste:
server {
    listen 80;
    server_name yourdomain.com www.yourdomain.com;

    root /var/www/tocken_app_Admin_panel/website/dist;
    index index.html;

    location / {
        try_files $uri /index.html;
    }
}

B) Admin panel config:
nano /etc/nginx/sites-available/admin.conf

Paste:
server {
    listen 80;
    server_name admin.yourdomain.com www.admin.yourdomain.com;

    root /var/www/tocken_app_Admin_panel/frontend/build;
    index index.html;

    location / {
        try_files $uri /index.html;
    }
}

C) API reverse proxy config:
nano /etc/nginx/sites-available/api.conf

Paste:
server {
    listen 80;
    server_name api.yourdomain.com;

    client_max_body_size 20m;

    location / {
        proxy_pass http://127.0.0.1:5000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

Enable all configs:
ln -s /etc/nginx/sites-available/website.conf /etc/nginx/sites-enabled/
ln -s /etc/nginx/sites-available/admin.conf /etc/nginx/sites-enabled/
ln -s /etc/nginx/sites-available/api.conf /etc/nginx/sites-enabled/

Test and restart Nginx:
nginx -t
systemctl restart nginx


STEP 7) Firewall (UFW)
----------------------
ufw allow OpenSSH
ufw allow 'Nginx Full'
ufw enable
ufw status

Note: port 5000 open karna zaroori nahi, API Nginx proxy se chalegi.


STEP 8) DNS Mapping
-------------------
Domain DNS panel me A records set karo:
- @      -> YOUR_VPS_IP
- www    -> YOUR_VPS_IP
- admin  -> YOUR_VPS_IP
- api    -> YOUR_VPS_IP

DNS propagation: 5 minutes to 24 hours.


STEP 9) SSL Setup (Certbot)
---------------------------
Install certbot:
apt install -y certbot python3-certbot-nginx

Get SSL certs:
certbot --nginx -d yourdomain.com -d www.yourdomain.com -d admin.yourdomain.com -d www.admin.yourdomain.com -d api.yourdomain.com

Test auto renew:
certbot renew --dry-run


STEP 10) Verify Deployment
--------------------------
Check PM2:
pm2 list
pm2 logs tocken-backend --lines 100

Check Nginx:
nginx -t
systemctl status nginx

Check API:
curl -I https://api.yourdomain.com

Open in browser:
- https://yourdomain.com
- https://admin.yourdomain.com

Quick Troubleshooting
---------------------
1) 502 Bad Gateway:
   pm2 logs tocken-backend
   pm2 restart tocken-backend

2) CORS/API issue:
   frontend .env -> REACT_APP_API_URL=https://api.yourdomain.com/api
   website .env  -> VITE_API_BASE_URL=https://api.yourdomain.com

3) React refresh 404:
   Ensure Nginx has: try_files $uri /index.html;

4) Mongo not connecting:
   Check MONGO_URI, Atlas DB user/pass, Network Access IP allowlist.
